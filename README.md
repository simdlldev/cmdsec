## 💡 How It Works

`cmdsec` acts as a safety assistant for your terminal:

1. **Risk Analysis**: It analyzes the given command or script, evaluating potential side effects, system modifications, disk wiping risks, or dangerous network activities.
2. **5-Level Color Verdict**: It returns a clear, color-coded safety assessment:
   - 🟢 **GREEN**: Safe standard command.
   - 🟡 **YELLOW**: Low risk (e.g. modifying user files).
   - 🟠 **ORANGE**: Execute with caution (potential side effects).
   - 🔴 **RED**: Potentially dangerous (data loss or service disruption).
   - 🟣 **PURPLE**: Destructive / Malicious (system wiping, malware, severe risk).
3. **Interactive Mode**: Launch `cmdsec` without arguments to paste or type multi-command strings (containing `&&`, `;`, `|`) without needing complex shell quotes.
4. **Automatic System Language**: Automatically switches between English and Italian based on your system language settings.

## 🚀 Installation

Clone the repository and run `install.sh`:

```bash
git clone https://github.com/simdlldev/cmdsec.git
cd cmdsec
./install.sh
```

During installation, you will be prompted to enter a free **Google Gemini API Key** (get one at [Google AI Studio](https://aistudio.google.com/app/apikey)).

Ensure `~/.local/bin` is in your `$PATH`. If needed, add this line to your `~/.bashrc` or `~/.zshrc`:

```bash
export PATH="$HOME/.local/bin:$PATH"
```

## 📖 Usage

### Interactive Mode (Recommended)
Run `cmdsec` without parameters to safely input any command:
```bash
cmdsec
```

### Direct Command Audit
```bash
cmdsec rm -rf /tmp/test
```

### Detailed Technical Explanation (`-a`)
```bash
cmdsec -a cat /etc/shadow
```

### Audit Script Files (`-s`)
```bash
cmdsec -s script.sh setup.sh
```

## 🗑️ Uninstallation

To remove `cmdsec` from your system:
